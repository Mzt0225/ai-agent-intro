## 一、论文写作的六大痛点

如果你写过论文获正在写论文，下面这些场景你一定不陌生：

### 😵‍💫 痛点一：文献综述——永远看不完的 PDF

打开 Google Scholar / CNKI，关键词一搜几百篇。精读十几篇之后，发现笔记散落在 PDF 批注、Word 文档、Notion 页面、微信收藏……等到动笔时，根本想不起来"那个关于工具变量的讨论是在哪篇文献里看到的"。

**结果**：要么文献综述写成流水账，要么遗漏关键文献被审稿人指出。

### 😶‍🌫️ 痛点二：写作启动困难——空白页恐惧症

引言怎么写？第一句怎么起？脑子里有想法，对着空白 Word 文档就是敲不出字。一拖就是一周，deadline 前熬夜赶工，质量可想而知。

### 🔁 痛点三：重复性体力劳动——清洗数据、逐条跑回归

实证研究的日常：从 CSMAR / CNRDS 下载原始数据 → Excel 手动整理 → 导入 Stata → 逐个变量生成交互项 → 一条条 `reg`、`xtreg`、`ivreg2` 复制粘贴 → 把结果手动摘到 Word 表格里。这些操作本身没有技术含量，但极其耗时且易出错。一个变量名写错，整条 do-file 重跑。

**结果**：大量时间耗费在"数据搬运工"和"代码复制粘贴"上，真正用于思考识别策略和理论逻辑的时间被严重挤压。

### 🧱 痛点四：实证软件门槛——Stata 劝退了多少好选题

Stata / R / Python 的学习曲线对初学者并不友好。安装包、配环境、调试 `r(198)` 报错……很多经管本科生和研一学生，有好想法但卡在软件操作上，不得不放弃需要稍复杂计量方法的选题。即使是有经验的实证研究者，每换一个新方法（比如从 DID 到 Bartik IV 到 SCM），依然要花大量时间搜教程、调代码。

**结果**：选题被工具能力限制，方法选择偏向"我会的"而非"最适合的"。

### 📏 痛点五：学术规范琐碎——格式比内容更磨人

参考文献格式（APA / GB/T 7714 / Chicago）、图表编号、公式排版、中英文对照……这些"体力活"占用了大量本应投入思考的时间。

**结果**：大量时间花在排版的活上，而非思考论证质量。更糟的是，格式问题往往是期刊初审退稿的第一理由——不是因为内容不好，而是因为"不规范"。

### 📭 痛点六：缺乏高质量反馈——导师太忙，同门不懂

写完初稿发给导师，一周后收到三条语音。同门水平相近，给不出建设性质疑。投稿被 desk reject，审稿意见寥寥几行，不知道问题出在哪。

**结果**：论文的致命缺陷——逻辑断点、识别漏洞、贡献模糊——长期得不到暴露，直到投稿被拒才被动发现，白白浪费数月时间。

## 二、为什么是 AI Agent，而不是聊天机器人

### 你用过 ChatGPT，但你可能没用过 AI Agent

| | ChatGPT / Kimi / 豆包（聊天式 AI） | AI Agent（如 Claude Code） |
|---|---|---|
| **交互方式** | 一问一答，手动复制粘贴 | 直接操作文件，读写执行一体化 |
| **上下文记忆** | 对话窗口有限，长文本易丢失 | 完整掌握项目文件夹，上下文持久 |
| **任务复杂度** | 适合单次问答 | 适合多步骤、多文件的复杂任务 |
| **工具调用** | 插件受限，功能割裂 | 自主搜索、读写文件、执行命令 |
| **工作流整合** | 需要在 AI 和写作软件间反复切换 | 在同一个环境中完成思考—写作—修改 |

### 一句话概括

> **聊天式 AI 是"顾问"——你问它答；AI Agent 是"合伙人"——它能在你的项目里直接干活。**

对于经管论文写作，这意味着什么？

- **文献综述**：Agent 可以批量阅读你 `literature/` 文件夹中的 PDF 笔记，自动提取核心论点、方法、数据来源，生成对比表格。
- **初稿撰写**：Agent 基于你的研究设计和数据描述，直接在你的 `.md` 或 `.tex` 文件中写出各个章节的初稿。
- **逻辑审查**：Agent 通读全文，检查"引言承诺—理论推导—实证结果—结论回应"的一致性，标出断点。
- **格式整理**：参考文献排序、图表编号、术语统一……Agent 一条指令搞定。

## 四、实战：用 Claude Code 完成一篇经管论文

以下是一个典型的经管论文写作流程，展示 AI Agent 在每个环节能做什么。

### 项目结构建议

```
my-paper/
├── main.tex (或 .md)          # 论文主文件
├── literature/                 # 文献笔记（每篇一个 .md）
├── data/                       # 数据描述与变量说明
├── results/                    # 回归结果表格
├── submission/                 # 投稿材料
└── CLAUDE.md                   # 项目说明（告诉 Claude 你的论文在做什么）
```

### 阶段一：选题与文献扫描

**你可以对 Claude Code 说：**

> "阅读 literature/ 文件夹下所有文献笔记，帮我做三件事：
> 1. 列出每篇的核心研究问题、方法、数据、结论
> 2. 识别这些文献之间的共识与争议
> 3. 标出我的研究可以填补的 gap"

Claude Code 会直接在你的项目中生成一个 `literature-summary.md`，包含结构化的文献对比表格。

**这替代了传统做法**：手动打开 30 篇 PDF → 来回切换窗口做笔记 → 花两天时间整理成 Excel → 写到论文里又忘记出处。

### 阶段二：研究设计

**你可以对 Claude Code 说：**

> "这是我的研究问题和理论框架（见 CLAUDE.md），请帮我：
> 1. 设计一个可行的实证策略，包括识别方法、数据来源建议
> 2. 写出计量模型设定的初稿
> 3. 列出需要控制的关键变量和理由"

Claude Code 会读取你的项目说明，结合文献综述中的常见做法，给出结构化的研究设计建议。

### 阶段三：撰写初稿

**这是 AI Agent 最亮眼的环节。你可以说：**

> "请根据以下内容撰写引言部分（2-3 页）：
> - 研究问题：XXX
> - 文献 gap（见 literature-summary.md 第三部分）
> - 边际贡献：三点
> 风格要求：《经济学（季刊）》风格，中文，正式但不晦涩。"

Claude Code 会：
1. 读取你的文献总结文件
2. 读取项目说明中的研究问题与贡献
3. 在你的 `main.md` 中写出引言初稿
4. 你可以接着让它修改、缩写、扩展某一段

**小节驱动写作法**：不要一口气让 AI 写全篇。逐节推进——引言 → 文献综述 → 理论框架 → 研究设计 → 实证结果 → 机制分析 → 结论——每节单独打磨，最后让 AI 通读全篇检查一致性。

### 阶段四：修改打磨

**你可以对 Claude Code 说：**

> "通读 main.md，检查以下问题：
> 1. 引言中声称的三点贡献，在结论部分是否都回应了？
> 2. 同一术语在全文中是否前后一致？（如'处理组/控制组' vs '实验组/对照组'）
> 3. 哪些段落过长、逻辑不清晰？标出具体位置并给出改写建议。"

### 阶段五：投稿准备

**你可以对 Claude Code 说：**

> "帮我准备《经济研究》的投稿材料：
> 1. 根据 main.md 生成中文摘要（300 字以内）
> 2. 生成英文摘要（对应翻译）
> 3. 列出需要附上的材料清单（如数据公开声明、作者贡献声明等）"

Claude Code 只是众多 AI Agent 工具中的一种。以下是当前主流选项的简要对比，**并非优劣排名，而是适用场景不同**：

- **项目级感知**：能否理解整个文件夹而不仅仅是当前打开的文件，是"聊天工具"和"Agent"的分水岭。

### 如果你符合以下情况，建议先从 Claude Code 入手：

- 主要任务是**论文写作**（而非写代码）

### 一个朴素的原则

> **工具是手段，思维是核心。**
>
> AI Agent 是你的写作放大器——它让你写得更快、改得更透、逻辑更清晰。但它不能替代你对研究问题的理解、对经济学的直觉、对数据质量的判断。**你仍然是论文的第一作者和最终责任人。**

## 七、常见问题与避坑指南

### ❓ AI 写出来的论文会不会查重过不了？

**正确使用方式不是让 AI 从零生成全篇**。AI Agent 的最佳角色是：
- 基于**你提供的研究设计、数据、文献笔记**来生成初稿
- 帮你**改写、精简、重组**你自己写的内容
- 生成的内容是你的研究、你的思想、你的数据——AI 只是帮助你表达得更好

此外，终稿必须经过你自己的修改和确认。AI 生成 + 人工深度修改，查重风险远低于直接复制粘贴其他论文。

### ❓ 能直接投稿 AI 写的内容吗？

不能，也不应该。原因有三：
1. **学术伦理**：大多数期刊要求作者对论文内容负责，AI 不能作为作者
2. **事实准确性**：AI 可能编造不存在的数据、引用、文献
3. **原创性要求**：论文的核心贡献必须来自研究者本人的思考和判断

**正确的做法**：AI 是辅助工具，你是有判断力的作者。AI 写初稿 → 你审查和修改 → AI 辅助打磨 → 你最终定稿。

### ❓ 如何避免 AI 编造参考文献？

这是一个常见陷阱。最佳实践：
1. **自己建文献库**：在 `literature/` 文件夹中手动录入你已读过的文献笔记
2. **指示 AI 只引用你的文献库**：在 CLAUDE.md 中明确说明"只引用 literature/ 中的文献"
3. **逐条核对**：终稿中的每一条参考文献，都必须在你自己的文献管理软件中确认过

### ❓ 多久能上手？

如果你有基本的技术素养（会用终端、了解文件路径），**一周内可以从零到能用**：
- 第 1 天：安装配置 Claude Code
- 第 2-3 天：用简单任务熟悉基本交互（改格式、写摘要）
- 第 4-7 天：开始正式用于论文写作

## 八、Claude Code 的 Git 集成与 Skill 生态

### Git：写作的安全网

上面已经提到，Git 集成让每一次 AI 修改都有据可查。具体到写作场景：

```
# 写作前：初始化 Git 仓库
git init
git add .
git commit -m "初稿：完成引言和文献综述"

# AI 修改后：查看改了什么
git diff main.md

# 改坏了？一键回退
git checkout -- main.md

# 大改方向不确定？开分支实验
git checkout -b ai-experiment
# ...让 AI 在分支上自由发挥...
git diff main  # 对比实验分支和主分支
git checkout main && git merge ai-experiment  # 满意就合并
```

对于一篇论文的写作周期（数月到一年），Git 记录的不仅是版本，更是你思考演进的完整轨迹。

### Skill：给 Claude Code 安装"专业插件"

Claude Code 支持通过 **Skill（技能）** 机制扩展能力。Skill 本质上是社区贡献的专业提示词包，安装后，Claude Code 在特定场景下会自动调用对应的专业知识。

#### 对经管论文写作特别有用的 Skill

| Skill 类型 | 用途 | 示例 |
|---|---|---|
| **学术写作规范** | 按特定期刊格式要求打磨论文 | 如按《经济学（季刊）》风格统一术语、格式 |
| **文献综述辅助** | 系统化整理和对比文献 | 自动生成文献对比矩阵、识别研究空白 |
| **图表设计** | 按学术出版标准优化图表 | 回归结果表的规范化输出 |
| **LaTeX 排版** | 公式、表格、交叉引用 | 自动修正 LaTeX 编译错误 |

#### 如何发现和安装 Skill

Claude Code 的 Skill 生态是**开源、社区驱动**的。安装 Skill 后，你可以：

1. **在 GitHub 搜索**：搜索关键词如 `claude-code skill economics`、`claude code 论文写作`，浏览社区贡献的 Skill 项目
2. **安装到本地**：下载 Skill 文件到项目目录的 `.claude/` 文件夹，Claude Code 会自动识别和加载
3. **自己定制**：Skill 本质是 Markdown 格式的提示词文件，你可以根据自己学科的需求修改甚至从头编写

```bash
# 典型的 Skill 安装：将下载的 skill 文件放入项目
# 社区 skill 通常只需放入 .claude/commands/ 或 .claude/skills/ 目录
mkdir -p .claude/skills
# 将下载的 skill 文件复制进去即可
```

## 九、推荐学习路径

### 第一步：建立认知（本文就够了）

### 第二步：上手实操

1. 安装 Claude Code（参考[官方文档](https://docs.anthropic.com/en/docs/claude-code)）
